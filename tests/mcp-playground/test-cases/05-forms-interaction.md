# 表单交互测试

测试 Browsion MCP 的表单操作功能。

## 前置条件

- Browsion Tauri 应用正在运行
- Claude Code 已连接到 MCP 服务器
- test-profile 已启动浏览器

## 测试用例

### 1. 填写表单

**在 Claude Code 中执行：**

```
请导航到 https://httpbin.org/forms/post，
然后在表单中：
- CustName 填入 "测试用户"
- CustEmail 填入 "test@example.com"
- CustTel 填入 "13800138000"
最后提交表单。
```

**预期结果：**
- 所有字段填充成功
- 表单提交触发

**验证点：**
- ✅ 字段值正确
- ✅ 提交成功

---

### 2. 选择下拉选项

**在 Claude Code 中执行：**

```
请在 httpbin.org 的表单中，
找到或创建一个 select 下拉框，
然后选择特定的选项。
```

**预期结果：**
- 下拉选项选中

**验证点：**
- ✅ 选择操作成功

---

### 3. 上传文件

**在 Claude Code 中执行：**

```
请创建一个包含文件上传的测试表单，
然后上传一个测试文件。
```

**预期结果：**
- 文件选择对话框触发
- 文件上传成功

**验证点：**
- ✅ 文件上传功能正常

---

### 4. 检查表单状态

**在 Claude Code 中执行：**

```
请告诉我表单的当前状态：
哪些字段有值？
哪些字段为空？
```

**预期结果：**
- 返回表单字段状态

**验证点：**
- ✅ 正确读取表单状态

---

## 测试总结

本场景测试表单交互功能：
- ✅ fill_and_submit - 填写并提交表单
- ✅ select_option - 选择下拉选项
- ✅ upload_file - 上传文件
- ✅ 表单状态读取

**表单操作技巧：**
- 使用元素定位器（selector）准确定位表单字段
- 检查字段是否可编辑（not disabled）
- 验证表单提交后的响应
